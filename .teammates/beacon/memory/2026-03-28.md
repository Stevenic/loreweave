## Task 5: NPC Companion & Offline Player Brainstorm
- Read full NPC System Spec (`npc_system_spec.md`) — 13 sections, 3-tier model, promotion system, §8 explicitly punts companions to post-v1.
- Cross-referenced existing types: `CompanionState`, `CompanionRelationship`, `CompanionReaction` (game.ts), `DisconnectEvent`, `ReconnectResult` (multiplayer.ts).
- Proposed 3 ideas for unifying NPC companions with offline player handling:
  1. **Autopilot Mode** — single `AutopilotController` pure function drives both NPC companions and offline players via `BehaviorProfile`. Offline players auto-stabilize (can't die).
  2. **Ghost Thread** — offline players become narrative-only (no mechanical participation). NPC companions are the separate mechanical system. Zero risk but weakens party.
  3. **Retainer Protocol** (recommended) — offline players literally become restricted `CompanionState` entries with `source: 'offline_player'` discriminator. Hard restrictions: can't die, can't spend consumables, can't make permanent choices. NPC companions use same type without restrictions.
- Recommended Idea 3 as strongest: single type system, stable encounter balance, explicit/auditable restrictions.
- No code changes — brainstorm only.
