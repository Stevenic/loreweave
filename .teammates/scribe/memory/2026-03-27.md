# Scribe — Daily Log — 2026-03-27

## Task 1: NPC System Design Proposal

- **What:** Designed the comprehensive NPC system architecture for LoreWeave — how NPCs spawn, persist, behave, and interact with players.
- **Context:** stevenic asked "how do we think NPCs work?" Current state has three disconnected layers: game types (Character/EntityRef — thin), world gen spawn rules (creatures only), and archetype templates (rich but no runtime model). The gap is connecting archetypes to living NPCs.
- **Key design decisions proposed:**
  - Three-tier model: Creatures (ephemeral, spawn rules), Residents (persistent, generated from archetypes), Named Characters (persistent, hand-authored by Bard)
  - `NPCRecord` type — persistent data with identity, traits, voice pattern, knowledge, disposition, schedule
  - LLM-generated dialogue — no dialogue trees, no stored lines. LLM gets NPC personality + knowledge + disposition + context and generates naturally.
  - Schedule-driven location — NPCs move between locations by time of day, creating natural scene variety
  - Disposition system — -100 to +100 toward player, gates trading, quests, and secret sharing
  - Skill-gated knowledge — Persuasion/Insight checks against DCs based on knowledge tier (always/sometimes/rarely)
  - NPC lifecycle: generated at settlement creation, persist forever, can die and be replaced via world healing rules
- **Open questions posed to stevenic:**
  1. NPC memory of past player interactions
  2. NPC-to-NPC relationships
  3. Companion NPCs (v1 or v2?)
  4. NPC count limits per settlement
  5. Trading system depth
- **Spec changes proposed (pending approval):** New `NPCRecord` type, new §8A in world gen spec (settlement NPC generation), expanded §4.4.3 (NPC context in narrative), new NPC system section in loreweave spec
- **Files changed:** None (proposal only — awaiting stevenic decisions)
