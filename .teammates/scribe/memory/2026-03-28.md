## Task 18: Character Creation Design Proposal
stevenic asked how character creation should work. Researched codebase — no creation flow exists, `Character` type is a minimal stat block (no class/race/background), only `standardArray()` and `rollAbilityScore()` exist as creation-adjacent code.

Proposed a 4-step narrative creation flow:
1. **Background** (6 options) — comes first because "who you are" is most narratively important in a text-driven game. Each background gives skill proficiencies, starting gear flavor, arrival hook, and NPC disposition modifiers.
2. **Class** (4 options: Fighter/Rogue/Cleric/Wizard) — maps to the 4 behavioral templates, covers all combat roles. Minimal v1 scope.
3. **Ancestry + Appearance** (4 ancestries: Human/Elf/Dwarf/Halfling) + free-text appearance description. Portrait generated via pixel LLM call (~64k tokens, cached permanently).
4. **Ability Scores + Name** — Standard Array or 4d6-drop-lowest, player assigns. Name is free text with optional LLM suggestions.

Key design decisions:
- Background first (not class) — most narratively relevant in LLM-narrated game
- Portrait is the one pixel LLM call in creation (aligns with three-tier cost model)
- V1 excludes: multiclassing, subclasses, feats, spell selection, re-spec
- Proposed `Character` type extensions: `class`, `ancestry`, `background`, `appearance`, `portraitAssetId`, `hitDie`, `armorProficiencies`, `weaponProficiencies`
- 4 open questions: deterministic rolls, portrait blocking vs async, quick start option, level-up flow
- **Files changed:** none (proposal text only, awaiting approval before writing spec)
