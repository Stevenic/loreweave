## Task 7: NPC Companion System Brainstorm
stevenic asked @everyone to brainstorm how NPC companions might work, with the constraint that the same system should handle offline players (who can't be killed). Proposed 3 designs:
1. **Companion as Tier 3 State Flag** — `companionState` overlay on Named Characters with `autonomyLevel: 'full' | 'restricted'`. Full combat participation, restricted mode for offline players (auto-stabilize at 1 HP, can't spend gold/accept quests). Same code path for NPC companions and offline surrogates.
2. **Retainer Model** — Companions provide a single bonus action (attack/Help/skill) on the player's turn instead of independent initiative. Simpler but less immersive.
3. **Campfire Circle** — Non-combat advisor companions who contribute knowledge, skills, and relationship access. Zero combat AI needed. Zero encounter rebalancing.
- **Recommendation:** Ship Idea 3 first (no combat AI, reuses existing dialogue system), architect data model for Idea 1 now.
- **Key D&D insight:** Any combat companion changes encounter math (10-25% more party damage output). Idea 3 avoids this.
- **Open questions:** Recruitment cost, per-player vs per-party cap, companion departure on low disposition.
- Handed off to Bard for narrative perspective.
