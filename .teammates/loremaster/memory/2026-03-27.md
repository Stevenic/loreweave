## Task 4: Post-Implementation D&D Accuracy Review

Reviewed all new engine code and content after Beacon's implementation push and Bard's archetype backfill.

### What Was Reviewed
- 5 new rules engine modules (encounter.ts, fray.ts, npc-generator.ts, skill-challenge.ts, ward.ts)

- 22 backfilled location archetypes (8 dungeons, 14 wilderness) with challenges field
- Fray Creature Catalog (fray_creatures.md)
- All type definitions in game.ts

### D&D Accuracy Issues Found

**Engine code (Beacon's domain):**
1. CR-to-XP table incomplete — stops at CR 8, uses `cr * 500` approximation for CR 9+ (should be: CR 9=5000, CR 10=5900, CR 11=7200, etc.)
2. Fray exposure save missing proficiency bonus — code does `d20 + CON mod` but D&D saves are `d20 + ability mod + prof bonus (if proficient)`
3. Skill challenge missing nat 20 = 2 successes (documented in Mechanics Companion §7)
4. NPC generator maps ALL authority NPCs to STR primary — magistrates/tax collectors should be CHA/INT, not STR
5. CreatureStatBlock type missing legendary actions, lair actions, reactions — needed for CR 9+ creatures
6. Expert tier proficiency bonus is +2 but covers CR 1-3 — should be +2 (correct per 2024 rules)
7. Elite tier prof +4 covers CR 9-20 but per D&D: CR 9-12=+4, CR 13-16=+5, CR 17-20=+6

**Content (Bard's domain):**
8. Location challenges mix saving throws with skill checks in the same array — "wisdom_save_resist_enchantment" and "dexterity_save_avoid_magical_surge" are saves, not skill checks
9. Fray Creature Catalog is excellent — no D&D issues found

## Task 5: Biome Coverage Assessment for Threadhallow

Reviewed engine's 7 biomes (forest, plains, mountain, swamp, tundra, desert, taiga) against Threadhallow's world content requirements.

### Assessment Summary
Current biomes cover basic terrain but miss several environments critical to the Threadhallow narrative:
- No coastal/shore biome (world mentions "opposite coast," dockside archetypes exist)
- No hills/moorland biome (Ashenmoor is a moor, Thornwick is in a valley between hills)
- No underground/cavern biome (Loom Chamber, cave systems, crypts)
- No farmland/pastoral distinction (villages surrounded by fields differ from wild plains)
- Forest lacks ancient/dense variant (Knotwood is narratively distinct from Greenmantle)

### Key Recommendation
Weave stability should be a MODIFIER on biomes, not separate biome types. The Frayed/Thinning/Unraveled states overlay any biome.

### Biomes Proposed
New: coast, hills, cavern, farmland, deep_forest (5 additions → 12 total)
Plus: weave_state overlay system (stable/thin/frayed/unraveled)
